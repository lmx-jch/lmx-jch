### Flare

I'm building [**Flare**](https://flare-ts.dev), a composition-first TypeScript HTTP framework for Node.js and Cloudflare Workers.

You register your config, services, routes, and contracts on one application graph, then call `build()`. That step validates the whole graph and compiles a pipeline for every route before a single request is served, so a broken wiring fails at startup instead of in production. It ships with zero third-party runtime dependencies, its own schema validation and router built in, and runs unchanged across runtimes by swapping one adapter.

The best way to see it is the [**sandbox**](https://sandbox.flare-ts.dev). Write a Flare app in your browser and run it live on Cloudflare. It builds as you type, you throw real requests at it, and it renders the compiled graph back to you. It's itself a Flare app, running on the same Cloudflare adapter it shows off.
